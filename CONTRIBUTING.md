# Contributing

## Workflow

1 Clone repo  
2 Create feature branch  
3 Implement changes  
4 Submit PR  

Example:

git checkout -b feature/my-improvement

## Branch Naming

feature/<topic>  
fix/<bug>  
docs/<documentation>  
experiment/<prototype>  

## Pull Requests

Include:

• change description  
• build status  
• reproduction steps  
• logs if relevant
